# 유튜브 광고 소재 비승인 예방 AI 솔루션 — 화면 기획서

> 사내 마케팅팀 전용 웹 어드민. 구글/유튜브 매체 등록 전, 광고 소재의 정책 위반 여부를 1차 검수하고 대체 문구를 제안하는 시스템.

---

## 0. 기획 요약

| 항목 | 내용 |
|---|---|
| 사용자 | 인하우스 마케터(검수 요청·확인), 관리자(키워드/프롬프트/데이터 관리) |
| 핵심 가치 | 비승인율 ↓, 광고 라이브 속도 ↑, 사전 셀프 검수 |
| 검수 3단계 | ① 룰베이스 키워드 매칭 → ② LLM 문맥 심사(확률 점수) → ③ RAG 유사 사례 보강 |
| 입력 | 영상 / 이미지(썸네일) 파일 |
| 추출 | Vision API(이미지 자막+좌표), Whisper(나레이션) |
| 출력 | 위반 위험 점수, 위반 문구 하이라이트, 대체 문구, UI 겹침 경고 |

전체 화면은 크게 **검수 영역**(마케터 일상 사용)과 **관리 영역**(데이터·정책 운영)으로 나뉩니다.

## 1. 정보 구조 (IA / 메뉴 트리)

```
로그인
│
├── 대시보드 (홈)            ← 검수 현황·통계 한눈에
│
├── 소재 검수
│   ├── 신규 검수 (업로드)    ← 핵심 진입점
│   ├── 검수 결과 상세        ← 핵심 작업 화면
│   └── 검수 이력             ← 과거 검수 목록/검색
│
├── 데이터 관리
│   ├── 금지 키워드 사전       ← 룰베이스 DB
│   └── 승인/비승인 히스토리   ← RAG 학습 데이터(엑셀 적재)
│
└── 설정 (관리자)
    ├── 가이드라인 / 프롬프트 관리
    ├── 사용자 / 권한 관리
    └── API / 시스템 설정
```

### 권한 정책

| 화면 | 마케터 | 관리자 |
|---|:--:|:--:|
| 대시보드 | ○ | ○ |
| 신규 검수 / 결과 상세 / 이력 | ○ | ○ |
| 금지 키워드 사전 | 조회 | 조회·편집 |
| 히스토리 데이터 적재 | △(요청) | ○ |
| 가이드라인/프롬프트 관리 | ✕ | ○ |
| 사용자/권한·시스템 설정 | ✕ | ○ |

## 2. 핵심 사용자 플로우

```
[1] 소재 업로드 (영상/이미지)
        │
        ▼
[2] 비동기 분석 (큐 등록 → 진행률 표시)
   ├─ Vision: 이미지 자막 + 좌표 추출
   └─ Whisper: 음성 → 텍스트 변환
        │
        ▼
[3] 1차 룰베이스 검수 — 금지 키워드 DB 매칭 → 하이라이트
        │
        ▼
[4] 2차 LLM 문맥 심사 — 위반 확률(%) + 사유
   └─ RAG: 과거 유사 사례 검색 → 프롬프트 주입
        │
        ▼
[5] 검수 결과 상세 화면
   ├─ 미리보기 + UI 겹침 경고
   ├─ 위반 문구 하이라이트 + 점수
   └─ 대체 문구 제안 (적용/복사)
        │
        ▼
[6] 마케터 최종 판단 (통과 / 보류 / 반려)
        │
        ▼
[7] 결과를 히스토리 DB 업데이트 → RAG 학습 루프 반영
```

상태 머신(검수 건 status): `업로드됨 → 분석중 → 검수완료 → (마케터 판정) 통과/보류/반려 → 매체등록완료`

## 3. 공통 레이아웃 규칙

```
┌──────────────────────────────────────────────┐
│  상단 바: 로고 │ 검색 │ 알림 │ 사용자(역할)       │
├────────┬─────────────────────────────────────┤
│        │                                       │
│ 좌측   │   본문 (페이지별 컨텐츠)                │
│ 메뉴   │                                       │
│ (IA)   │                                       │
│        │                                       │
└────────┴─────────────────────────────────────┘
```

- 좌측 고정 GNB(접기 가능), 상단 글로벌 바, 본문 영역.
- 모든 비동기 작업은 토스트 + 진행률로 피드백. 페이지 이동해도 백그라운드 처리 유지.
- 공통 상태 4종을 모든 데이터 화면에 정의: **로딩 / 정상 / 빈 값(Empty) / 에러**.
- 위험도 색상 토큰: 🟢 안전(0–30%) · 🟡 주의(31–70%) · 🔴 위험(71–100%).

## 4. 화면별 상세 기획

### 4-1. 로그인 / 접근 인증

| 구분 | 내용 |
|---|---|
| 목적 | 사내 마케팅팀 전용 접근 제어 |
| 구성 | 사번/이메일 + 비밀번호, SSO(사내 계정 연동 권장), 2단계 인증(선택) |
| 요소 | 로그인 폼, 에러 메시지, "관리자 문의" 링크 |
| 상태 | 인증 실패 / 권한 없음 / 세션 만료 안내 |
| 비고 | 광고 소재 = 민감 데이터이므로 세션 타임아웃·접근 로그 기록 |

### 4-2. 대시보드 (홈)

마케터가 로그인 후 처음 보는 현황판.

```
┌─────────────────────────────────────────────┐
│  [이번 달 검수 건수]  [예상 비승인 차단]        │  ← KPI 카드 4종
│  [평균 위험 점수]     [평균 처리 시간]          │
├──────────────────────┬──────────────────────┤
│  비승인 위험 추이(라인) │  위반 유형 분포(도넛)   │
├──────────────────────┴──────────────────────┤
│  최근 검수 내역 (테이블: 소재명/유형/점수/상태)   │
│  [+ 신규 검수 시작] 버튼                        │
└─────────────────────────────────────────────┘
```

| 요소 | 설명 |
|---|---|
| KPI 카드 | 기간 내 검수 건수, 위험 차단 건수, 평균 위험 점수, 평균 분석 소요 시간 |
| 추이 차트 | 일/주 단위 위험 점수·검수량 라인 차트 (기간 필터) |
| 위반 유형 분포 | 오도성/과장/금지어/UI겹침 등 카테고리 비율 |
| 최근 내역 | 클릭 시 결과 상세로 이동 |
| 상태 | 데이터 없을 때 "첫 소재를 검수해 보세요" Empty + CTA |

---

### 4-3. 신규 검수 (업로드)

핵심 진입 화면.

```
┌─────────────────────────────────────────────┐
│   소재 파일을 끌어다 놓거나 선택하세요             │
│   ┌───────────────────────────────────┐     │
│   │   ⬆  Drag & Drop  (영상/이미지)      │     │
│   └───────────────────────────────────┘     │
│   캠페인명 [____]  매체 [유튜브 ▾]  메모 [__]   │
│                                               │
│   업로드 목록 (파일명 / 용량 / 진행률 / 상태)     │
│                          [검수 시작]            │
└─────────────────────────────────────────────┘
```

| 요소 | 설명 |
|---|---|
| 업로드 영역 | 드래그앤드롭 + 멀티 파일. 지원 포맷·최대 용량 안내 |
| 메타 입력 | 캠페인명, 매체, 광고 형식(범퍼/인스트림 등), 메모 |
| 진행률 | 대용량 영상은 청크/비동기 업로드 + 진행 바 |
| 처리 안내 | "분석은 백그라운드 진행, 완료 시 알림" |
| 상태 | 포맷·용량 초과 에러, 업로드 실패 재시도 |
| 데이터 흐름 | 업로드 → 스토리지 저장 → 분석 큐 등록 → 결과 상세로 이동 |

### 4-4. 검수 결과 상세 ★ (가장 중요한 화면)

추출·검수·제안 결과를 한 화면에서 검토하고 판정.

```
┌───────────────────────────┬─────────────────────────┐
│  [미리보기]                 │  [검수 결과 패널]          │
│  영상 플레이어 / 이미지       │  종합 위험 점수  🔴 82%    │
│  └ 유튜브 하단 UI 가이드 오버레이│  ──────────────────     │
│  └ 자막 좌표 박스 표시        │  ① 룰베이스: 금지어 2건    │
│                            │  ② LLM 문맥: 오도성 높음   │
│  [타임라인 / 장면 썸네일]     │  ③ 유사 사례: 3건(비승인)  │
│                            │  ──────────────────     │
│                            │  탭: [추출텍스트][위반항목] │
│                            │       [대체문구][유사사례]  │
├───────────────────────────┴─────────────────────────┤
│  하단 액션 바:  [통과] [보류] [반려] │ 코멘트 입력         │
└───────────────────────────────────────────────────────┘
```

좌측 — 미리보기 & 시각 검증
- 영상: 플레이어 + 장면 썸네일 타임라인. 이미지: 확대/축소.
- **UI 겹침 경고**: 유튜브 하단 UI(제목/CTA/진행바) 영역을 반투명 가이드로 오버레이 → 자막 좌표가 겹치면 🔴 표시.
- Vision으로 추출한 자막 위치를 바운딩 박스로 표시.

우측 — 검수 결과 (탭 구성)

| 탭 | 내용 |
|---|---|
| 추출 텍스트 | 이미지 자막(좌표 포함) + Whisper 나레이션 텍스트(타임코드). 원문 보기/복사 |
| 위반 항목 | 룰베이스 매칭 키워드 하이라이트 + LLM 판정 사유. 항목별 위험도 배지 |
| 대체 문구 제안 | 위반 문구별 심의 통과 가능 대안 1~3개. [적용]/[복사], 사유 설명 |
| 유사 사례 (RAG) | 과거 승인/비승인 유사 소재 + 유사도(%) + 당시 결과/사유 |

| 요소 | 설명 |
|---|---|
| 종합 점수 | 0~100% 위험 확률 + 색상. 신뢰도/판정 근거 요약 |
| 액션 | 통과/보류/반려 + 코멘트. 판정 즉시 히스토리 DB 반영(학습 루프) |
| 상태 | 분석중(부분 결과 스트리밍 표시), 일부 API 실패 시 해당 섹션만 에러+재시도 |
| 비고 | 결과 PDF/링크 공유로 팀 검토 가능 |

### 4-7. 승인/비승인 히스토리 (RAG 학습 데이터)

| 요소 | 설명 |
|---|---|
| 데이터 적재 | 과거 승인/비승인 소재 엑셀 업로드 → 임베딩 → Vector DB 적재 |
| 적재 현황 | 총 건수, 승인/비승인 비율, 마지막 적재일, 임베딩 처리 상태 |
| 목록 | 소재 텍스트, 결과(승인/비승인), 사유, 카테고리, 출처 |
| 재인덱싱 | 임베딩 모델 변경 시 재처리, 진행률 |
| 피드백 루프 | 신규 검수 판정 결과가 자동으로 데이터셋에 누적됨을 표시 |
| 상태 | 적재 실패 행 리포트, 대용량 처리 비동기 진행 |

## 6. 우선순위(MVP) 제안

| 순위 | 화면 | 비고 |
|---|---|---|
